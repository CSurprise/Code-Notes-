HTML-Hyper Text Markup Language

The language used to create the web pages you visit everyday. (It is a logical way to structure content for webpages.)

A Markup Language is a computer language that defines the structure and presentation of raw text

Standardized Vocab

  1. Angle Brackets- In HTML, the characters < and > are known as angle brackets
  2. Element- The code that lives inside of the < > angle brackets
  3. Opening Tag- The first opening tag to start an HTML element
  4. Closing Tag- The second (or closing) tag used to end an HTML element closing tags have a forward slash inside of them.

    5. Example: Opening "html" Closing "/html"

  5. Attribute- Provides even more info. about an elements content. They live in the opening tag of an element.

    6. Attributes are made of two things, the value and the name of the attribute

  6. Hyperlink- A link
  7. Type- Describes the document you are linking to. Ex. Value=text/css
  8. rel- Describes the relationship between the 2 files, (If you are linking to a style sheet like CSS the value is stylesheet) Ex. rel=stylesheet
  9. UX= User-Experience
  10. Foreground- the element.
  11. Background- behind the element
  12. Typography- The art of writing
  13. Serif: The letters in these fonts have extra details on the ends of each letter and traditionally used in print designs.
  14. Sans-Serif: Letters do not have extra details just straight flat edges. This looks better on low-resolution screens
  15. Px- Represents the unit of pixels. A pixel is a small point on a screen and most monitors have a resolution of 72 pixels per inch. Making a pixel 1/72 of an in.
    p {font-size: 18px;}
  16. em- pronounced just as it looks, "em". A em is equal to the width of the letter m. Ems are a relative unit of measurement. They change the size of text relative to the parent element's size of text
    p {font-size: 1.3em;}
  17. %- Based on the default size of pixels in a browser
    p {font-size: 150%;}
  18. ID- Intended to label unique elements in an HTML file. No two HTML elements should ever share the same ID

HTML Elements

Heading sizes

There are multiple heading sizes that can be used on a webpage in HTML. They range from H1 to H6 from largest to smallest

Headers are usually used to grasp a readers attention with a few words. If you want to put words in paragrapgh format use "p". This is great for expanding the amount of words on your webpage.

Lists

Often times you will want to display content in a list. Im HTML you will want to use an unordered list "ul" for bullet points or an ordered "ol" for numbered

However these elements cannot hold raw text and cannot format text because of this

Individual list items must be added to the list by using the listed item element "li"

Hyper-Links

you can add links to a webpage by adding the anchor element "a" with URL in between the opening and closing tag

However! You can not use the anchor element and expect the link to work without an attribute. The href attribute

For anchor elements (a hyperlink), name of the attribute is href and it's value must be set to the URL of the page you'd like the user to visit

Example: "a href=https://www.wikipedia.org" this is wikipedia! "/a"

For a link that leads to a different window/tab, we use the target attribute.

The target attribute requires a value of _blank . Like all attributes this should be in the opening tag of your anchor element

For example: "a href=https://www.wikipedia.org target=_blank" Wiki! "/a"

Pictures

If we wanted images? we would use the image element "img/" this element is special because it is self closing

The image element requires the element src (sort of like href) with the value = to the URL of the picture

"img src=https://picture.url /"

Visually impaired

HTML supports the visually impaired with the alt attribute

The alt attribute is applied specifically to the img element and it's value should be a descripton of the picture

An example would be : "img src=https://SpidermanPicture alt= a picture of spiderman /"

If the image provides no important info. then an alt should not be applied

Hyperlinking with Images

If we want to add a hyperlink to an image we must put the image element INSIDE of the anchor element

For example: "a href=URL target=_blank" "img src=PictureUrl alt=Description /" "/a"

CSS Coding

With the basics of html that we now possess the website will still very plain and boring

To make websites more visually appealing we use CSS (Cascading Style Sheets) a language used to style HTML content. CSS is used to modify:

  • colors
  • font type
  • font size
  • shadows
  • images
  • element positioning
  • Et cetera

Even though CSS is a different language, it is still possible to use in HTML due to the "style" element

The Style Element

This element allows you to write CSS code between its opening and closing tags

The style element must be inside the "head", once "style" is placed inside of the web page's head, we can begin writing in code

However, this mixing of code is very confusing and can result in the risk of 1 or 2 things

  1. Creating a larg HTML file that is both difficult to read and maintain. This could result in inefficient work flow
  2. Maintaining a clear distinction between web page structure (html) and web page aesthetics

Styling with a CSS file

To fix having too much clutter we can seperate the styling (CSS) from the index.html file to the style.css file.

However just putting code in a seperate file will not work, the index file must be linked with style.css

In order to do this we use the "link" element. This has to be put in the head of the HTML file, it is a self closing tag and REQUIRES href (like the anchor) and the value is = to to the address or path to the CSS file.

Linking

When linking a html and css file together the "link" element will look lke:

"link href=/style.css type=text/css rel=stylesheet"

Actually Styling

To first start styling we have an html element we must select the element in the css profile

For example to style a "p" element the syntax in the css would be p { }

In this example ALL paragraph elements are selected. The seletor is the HTML tag (in this case p) but without angle brackets.

To actually style the element you need to specify two things.

  1. Property- The property you would like to style of the element (size, color, font size, etc.)
  2. Value- The value of the property (18px for size, Blue for color, etc.)

h1 {color:Blue;}

h1 is the selector, color is the property, Blue is the value, and the semicolon is used to end the declaration

The line "color:Blue" is a declaration. Declarations consist of a property and a value.

The whole line of code is known as a CSS Rule. A CSS rule consists of the selector and all declarations in the selector.

Grouping

If multiple elements would like to be selected for the same changes then simply add them to the selector... h1, p {color:Blue;}

When all elements on a webpage require the same styling, it is often more efficient to set the styling using the universal selector *, Afterwards you can modify (or remove) that styling for specific elements that don't require it

Commenting in CSS

Just like HTML you can leave comments in a CSS file

/* comment */

This will accomplish the samething html commenting does.

Colors

There are 2 distinctions between what color will affect.

  1. The foreground color.
  2. The background color

In CSS, these 2 designs can be styled by:

  1. color: This styles an elements foreground.
  2. background-color: This styles an elements background

Ex: h1 {color:Red;
background-color:Blue;
}

This is the appropriate way to code in CSS (even though it looks odd)

There are 147 named colors that can be used just google them.

RGB Colors

Even though there are only 147 full colors we would not be taking full advantage of color without using RGB

RGB(Red,Green,Blue) offers the option of 16,777,216 million possible colors

Each color in RGB can be one of 256 possible values. (They range from 0 to 255)

Ex. h1 {color:rgb(123,20,233);}

Hex Color

You can also use hex color codes but they are basically the same thing as RGB just a different way to write color i.e. h1 {color: #AA33BB or #452F73}

HSL

A new way to specify colors is through HSL coloring.

  • Hue: The technical term for color. In HSL, hue is represented on a color wheel. It can have values from 0-360
  • Saturation: The amount of gray in a given color. In HSL, saturation is specified using percents between 0% to 100%, 0% being a very light shade of gray and 100% being full saturation
  • Lightness: The amount of white in a given color ranging from 0% to 100%. 0% being black, 50% normal, and 100% white

    • For example {color: hsl(182,20%,50%)}

Opacity

Opacity is the measure of how transparent a color is.

RGB is very simialar to hex color except RGB can add opacity to it by doing rgba() instead of just rgb()

The a character in rgba is known as the alpha value. It represents the opacity of color. The value ranges from 0-1 inclusively

h1 { color:rgba(123,88,9,.5);}

the .5 represents 50% opacity which is normal color.

Opacity can also be used with HSLA

Font

The phrase "Type of Font" refers to the technical term typeface, or font family.
To change the type of font we use:

Example: h1 {font-family:Garamond;}
name of the font is always capital and when using a font with more than one word the words are in quotation i.e. "Times New Roman"

When setting new font it is important to keep this in mind

  1. The font specified must be installed on the user's computer in order for the font to display.
  2. the default typeface for HTML is Times New Roman.
  3. It is good practice to limit the number of typefaces to 2 or 3 on a site.

Like color, sometimes a browser cannot open certain fonts and fallback fonts are needed. Most computers only hsve a few pre-installed fonts.

To use fall-back fonts, the following syntax is required:
h1 {font-family: Garamond, Times, serif;}

The rule above states to use Garamond font, if that is not availabe then uses the Times font, if that is not availabe use any serif font pre-installed on the computer

Luckily new fonts are constantly being centralized in directories available for public use.

For example: With google fonts you can use the URL for a certain font and link it to make it available to use.

"head"
"link href=font.url type=text/css rel=stylesheet" "/head"

This makes the font of the URL available to use in CSS as you normally would

Font Size

Another way to customize font is through it's size.
p {font-size: 18px;}

In the example font size is set to 18px. px is a unit of measurement such as ft or in. px is for pixels

Line Spacing

To avoid words being too closely cluttered together and hard to read we have "line-height" to modify the spacing between lines of text
p {line-height:1.5em;}

When the line height of an element is modified, you are manipulating the leading (pronounced ledding) of the font. When the line height is increased, the spacing between lines of text increases, which can make text easier to read.

Line height can be modified with px or ems but ems are preferred

Spacing in general

You can also space words or letters in the same way as line spacing by using the "word-spacing" or "letter-spacing" properties

The default space between words in .25em but it is not common to increase the length

Also changing the length of letter spacing is called kerning

Text

A common practice found on websites is bolding.

In CSS the "font-weight" property turns boldness on or off
p {font-weight: bold;} or p {font-weight: normal;}

We can also change boldness using numbers starting from 100 every 100 to 900

You can also italicize words with the font-style property
h3 {font-style: italic or normal;}

Or you can change letters from all uppercare or lowercase with the text-transform property
h1 {text-transfrom:uppercase or lowercase;}

Text will always appear on the left side but to be moved use "text-align"
h1 {text-align:left center or right;}
